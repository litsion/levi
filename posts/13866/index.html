📝记录系统重装后Blog的迁移问题

发表 - 2019-06-22 分类 - 个人记录
就是一个小萌新
重装Blog后处处碰壁的悲伤故事
已哭晕在电脑桌前

✏️ 问题总结

Git Bash 后 弹出 bash: hexo: command not found

之前备份的Blog，根目录Git Bash发现弹出来 bash: hexo: command not found
好嘛这就很简单了，HEXO的变量挂掉了
重新再系统里设置下变量就好了，在Path里添加博客node_modules下的.bin文件路径
Git Bash 后输入 hexo s ，本地浏览成功，搞定

重新部署至Github，还好我有备份密匙 😎

安全起见我还是先弄个新的密匙吧~

git config --global user.name "user_name"
git config --global user.email "user_name@mail.com"

执行完后进入隐藏文件夹

cd  ~/.ssh

回车3次，生成密匙

ssh-keygen -t rsa -C "user_name@mail.com"

再敲下代码，密匙文件放在在User/.ssh里

eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_rsa

因为我有备份绑定在Github的密匙了，所以我只要把旧的密匙覆盖到.ssh的目录里，再从Blog的文件里Git Bash一下，就连接成功了

ssh -T git@github.com

回车完后弹出了Hi，Litsion的字样，真香

清理下项目

hexo clean

重新生成

hexo generate

部署

hexo deploy

本地预览发现js特效全线阵亡

不知道咋整
后来重新搭建了HEXO
再安装我喜欢的Skapp主题包
还是一加载网页js特效就阵亡
难过
Skapp换成了Next.Muse 完美运作了
心里特别委屈
